# RPG Project: DEUS CODE
## デウス・コード 八百万の神託

これは、AIと日本神話を融合させた現代ファンタジーRPG「デウス・コード」のゲームデザインドキュメント（GDD）を管理するリポジトリです。

### コンセプト
「神々の力をその身に降ろし、AIが支配する無機質な世界に、人の心を取り戻せ」

### 含まれる内容
- シナリオ生成用データ
- 各種アセットの画像生成用プロンプト
- プレイアブルなRPGデモ（HTML5/Canvas）

### プロジェクト構成
```
deus-code-gdd/
├── README.md               # プロジェクトの概要
├── game.html               # エントリポイント（src/ を順番に読み込む）
├── src/                    # ゲーム本体（骨格＝ロジック と 描画 を分離）
│   ├── constants.js        #   定数・canvas/ctx
│   ├── util.js             #   汎用ヘルパー
│   ├── assets.js           #   画像アセット読み込みフック（field.png 等）
│   ├── state.js            #   gameState / newGame / ステータス同期
│   ├── save.js             #   セーブ・ロード（localStorage）
│   ├── input.js            #   キーボード入力ルータ（シーン別）
│   ├── main.js             #   ゲームループ・起動
│   ├── data/               #   データ駆動の中身（内容追加で物語が伸びる）
│   │   ├── skills.js  items.js  enemies.js  characters.js
│   │   ├── shops.js   maps.js   chapters.js  cutscenes.js
│   ├── systems/            #   純ロジック（状態遷移・進行・戦闘・メニュー）
│   │   ├── cutscene.js story.js world.js battle.js
│   │   ├── menu.js     title.js scene.js
│   └── render/             #   描画層（差し替え可能：ここを作り込めば見た目が完成）
│       ├── common.render.js   cutscene.render.js  world.render.js
│       ├── battle.render.js   menu.render.js
├── data_for_claude/
│   └── scenario_data.md    # Claudeに渡す、シナリオ生成用の全データ
└── image_prompts/          # 画像生成AI用のプロンプト集
    ├── 01_characters.md  02_enemies.md  03_environments.md
    ├── 04_items.md       05_ui.md       06_animations.md
```

#### アーキテクチャ方針：ロジックと描画の分離
ゲームループは `updateScene()`（ロジック）→ `renderScene()`（描画）の2段構成。
進行・戦闘・メニュー等の**骨格はすべて `systems/` と `data/` で完結**し、Canvas描画は
`src/render/` の薄い層に切り出している。**見た目の作り込みは `render/*` の各関数の中身を
差し替えるだけ**でよく、ロジックには手を入れずに済む（描画の担当を分けられる設計）。

### ゲームをプレイする
`game.html` をブラウザで開いてプレイできます（ビルド不要）。

**操作方法:**
- ↑←↓→: 移動 / 選択
- SPACE（または Enter）: 会話・決定・カットシーン送り
- X: メニュー（アイテム / 神威 / そうび / ステータス / セーブ）
- S: クイックセーブ

**ゲームの流れ（ソフト誘導）:**
タイトル → オープニング（親友の"処理"→スサノオ覚醒）→ 新宿ハブ。
画面に常時「目的」が表示され、対象の場所/人物に近づくと自動でイベントが発生。
第1章（覚醒）→ 神社でアカリ → 闇市でヤミ → 植物園でリク → 都庁潜入 → アーク中枢、
と仲間を集めながら物語が自然に進行する。

**実装済みの機能:**
- タイトル → オープニング → マップへの滑らかな導入（カットシーン＋暗転演出）
- 章・クエスト・ストーリーフラグによる進行管理（ソフト誘導の目的表示）
- 進行に応じて変化するNPC会話／仲間の順次加入（アカリ・ヤミ・リク）
- ターン制バトル（たたかう／神威／どうぐ／にげる、EXP・レベルアップ・神威習得）
- メニュー：インベントリ・装備・ステータス（スキルツリー）・セーブ
- ショップ／宿屋（売買・HP/MP回復）
- セーブ／ロード（localStorage・「つづきから」・章完了時オートセーブ・神社の老巫女に話してセーブ）
- マップ再構成（神社境内・植物園・アーク中枢を追加）と論理的なマップ間移動
- バランス調整済み（各章ボス戦が10〜25ターンで決着するよう自動シミュレーションで検証）
- クリア後フリーローム＋メニューから「タイトルへ戻る」

**設計上のポイント:**
1. ✅ オープニング→マップが断絶しない自然な導入に再構成
2. ✅ ロジック（systems/・data/）と描画（render/）を分離 ＝ 描画だけ差し替え可能
3. ✅ データ駆動（章・カットシーン・敵・アイテム・神威・マップ）で拡張が容易

### その他の使用方法
1. `data_for_claude/scenario_data.md` の内容をClaudeに渡すことで、ゲーム設定に基づいたシナリオを生成できます
2. `image_prompts/` 内の各ファイルには、画像生成AI用のプロンプトが含まれています
